## 规则 0：最低数据资格

只有在分析期间内拥有 **>=50 次转化** 的渠道才有资格进行预算调整。

低于阈值的渠道：分类为 **INSUFFICIENT_DATA（数据不足）-> MAINTAIN（维持）**。

---

## 规则 1：渠道分类

将每个渠道精确归入一个类别。按**顺序**应用规则（首先匹配的规则优先）：

### PAUSE（暂停，减少 100%）
满足任一条件即触发：
- 用户表示该渠道已连续 3 周以上处于负状态
- 用户表示经过 3 周以上优化努力后仍无改善

### DECREASE_HEAVY（大幅减少，减少 45%）
满足任一条件即触发：
- ROAS < 目标值的 50% 且净利润 <= 0
- CPA > 最大值的 150% 且净利润 <= 0
- 三项全部失败：ROAS < 100% 且 CPA > 100% 且净利润 <= 0
- 用户表示该渠道已连续 2 周处于负状态

### INCREASE（增加，使用用户指定上限，默认 15%）
必须同时满足所有条件：
- ROAS >= 目标值的 115%
- CPA <= 最大值的 80%
- 净利润 > 0

### DECREASE_LIGHT（轻度减少，减少 25%）
满足任一条件即触发（若尚未被分类）：
- ROAS < 目标值的 80%
- CPA > 最大值的 120%

### MAINTAIN（维持，变化 0%）
不满足上述任何条件。

## 规则 2：计算预算变化

### 第一步：计算减少量（全额应用）
```
DECREASE_HEAVY：减少量 = 当前支出 × 0.45
DECREASE_LIGHT：减少量 = 当前支出 × 0.25
```

### 第二步：计算释放预算
```
freed_budget（释放预算）= 所有减少量之和
```

### 第三步：分配给 INCREASE 渠道
按净利润按比例分配：
```
weight（权重）= 渠道净利润 / 所有 INCREASE 渠道净利润之和
proposed_increase（建议增加量）= freed_budget × weight
```

### 第四步：应用上限

**单渠道上限：** 若用户已指定值则使用用户值，否则默认为 15%。
```
max_increase（最大增加量）= 当前支出 × increase_cap（增加上限）
final_increase（最终增加量）= min(proposed_increase, max_increase)
```

**用户重新分配限额**（若已指定）：
- 仅适用于**增加**，不适用于减少
- 若 sum(proposed_increases) > user_limit：按比例缩减增加量
```
scale_factor（缩放系数）= user_limit / sum(proposed_increases)
final_increase = proposed_increase × scale_factor
```

### 第五步：计算未分配节省额
```
unallocated（未分配）= freed_budget - sum(final_increases)
```
报告为"可用于储备"。

## 多周调整

若用户提供历史背景信息，调整分类：

| 用户所说 | 调整方式 |
|---------|---------|
| "渠道 X 已连续 2 周以上处于负状态" | 升级为 DECREASE_HEAVY 或 PAUSE |
| "渠道 X 已连续 3 周以上处于负状态" | PAUSE（预算设为 $0） |
| "我们上周更改了渠道 X 的预算" | 覆盖为 MAINTAIN（允许 5-7 天稳定期） |
| "渠道 X 较上周有所改善" | 可将 DECREASE_LIGHT 升级为 MAINTAIN |

若无历史背景信息，仅使用单周分类。

---

## 决策矩阵

| 条件 | 分类 | 变化 |
|-----|------|------|
| 连续负 3 周以上（用户陈述） | PAUSE | -100% |
| 3 周以上无改善（用户陈述） | PAUSE | -100% |
| 连续负 2 周（用户陈述） | DECREASE_HEAVY | -45% |
| ROAS < 50% 且净利润 <= 0 | DECREASE_HEAVY | -45% |
| CPA > 150% 且净利润 <= 0 | DECREASE_HEAVY | -45% |
| ROAS < 100% 且 CPA > 100% 且净利润 <= 0 | DECREASE_HEAVY | -45% |
| ROAS >= 115% 且 CPA <= 80% 且净利润 > 0 | INCREASE | +15% 上限 |
| ROAS < 80% | DECREASE_LIGHT | -25% |
| CPA > 120% | DECREASE_LIGHT | -25% |
| 所有其他情况 | MAINTAIN | 0% |

## 必需输出格式

### 1. 分类表
| 渠道 | ROAS | 目标占比 | CPA | 最大值占比 | 净利润 | 分类 |

### 2. 计算步骤
显示：释放预算、分配权重、建议增加量与上限增加量对比、未分配节省额。

### 3. 最终重新分配表
| 渠道 | 当前预算 | 变化 | 新预算 | 分类 |

若未分配 > 0，则包含"储备"行。
